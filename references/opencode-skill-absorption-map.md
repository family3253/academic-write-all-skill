# OpenCode Skills Absorption Map for cycwrite

## Directly Relevant Skills

### Core paper-writing skills
- `academic-paper`
  - absorb: configuration interview mindset, multi-mode paper writing, bilingual abstract awareness, citation compliance expectations, format-convert awareness, figure-aware writing, revision-coach style output patterns
  - do not absorb wholesale: its multi-agent execution protocol, venue-specific LaTeX hard requirements, and full orchestration complexity

- `academic-paper-reviewer`
  - absorb: multi-perspective review thinking, methodology vs domain vs devil's-advocate separation, structured editorial decision logic, revision roadmap framing, re-review mindset
  - do not absorb wholesale: complete 7-agent simulation architecture

- `academic-pipeline`
  - absorb: stage detection, mid-entry awareness, mandatory integrity boundaries, review/revise/re-review/finalize state thinking
  - do not absorb wholesale: full state machine and pipeline enforcement, because cycwrite should remain a writing super-skill, not a heavyweight orchestrator

### Review / evidence synthesis skills
- `cross-disciplinary-review-writer`
  - absorb: review-type routing, evidence gates, framework selection, corpus-first review writing logic, downgrade rules when evidence is insufficient, and lightweight project-artifact thinking (candidate pool, screening records, evidence extraction tables)

- `deep-research`
  - absorb conceptually: research-question clarification, fact-check awareness, source verification expectation
  - not absorbed operationally unless user wants a research workflow rather than writing workflow

- `research-lit`
  - absorb: multi-source literature discovery mindset, local-library-first awareness, related-work table outputs, and narrative synthesis from paper tables

- `arxiv`
  - absorb: explicit arXiv retrieval awareness for frontier CS/AI topics, structured metadata pull, and preprint-vs-peer-reviewed distinction

- `pubmed-database`
  - absorb: stronger PubMed / MeSH / field-tag / E-utilities literacy for biomedical review-grade search planning

### Citation / compile / format / venue skills
- `citation-management`
  - absorb: DOI verification, metadata normalization, deduplication awareness, structured citation outputs

- `latex-compile-qa`
  - absorb: LaTeX build-failure awareness, compile-chain consciousness, not writing raw TeX claims without compile expectations

- `venue-templates`
  - absorb: venue-specific structure and formatting sensitivity

### Figure / table / document-adjacent skills
- `paper-figure`
  - absorb conceptually: publication-grade figure planning and figure-driven manuscript support

- `scientific-visualization`
  - absorb: figure planning as part of manuscript quality, not as a decorative afterthought

- `scientific-schematics`
  - absorb: mechanism / workflow diagram planning in reviews and papers

### Writing-quality skills
- `document-skills`
  - absorb: clarity, flow, long-form document structuring

- `markitdown`
  - absorb: markdown normalization awareness for manuscript source handling

## cycwrite Capability Upgrades To Add

1. **Mode expansion**
   - add explicit `Paper-Workflow Awareness`
   - add `Re-review / verification review` concept
   - add `Venue-fit / format sensitivity`

2. **Reference integrity expansion**
   - DOI verification and metadata completion awareness
   - deduplication and export-oriented citation handling awareness

2b. **Literature search projectization**
   - lightweight artifact stack for candidate pool, title/abstract screening, full-text screening, and evidence extraction
   - PRISMA / PRISMA-S style reporting awareness without pretending full compliance

3. **Review intelligence expansion**
   - distinguish methodology review, domain review, perspective review, and devil's-advocate review
   - support editorial decision and revision roadmap outputs

4. **Manuscript lifecycle expansion**
   - draft -> integrity -> review -> revise -> re-review -> finalize
   - keep as conceptual routing, not full pipeline enforcement

5. **Visualization-aware writing**
   - manuscript sections should respect figures, legends, and evidence displays
   - review writing should include table/figure planning where helpful

## Deliberate Non-Absorption

These should remain outside cycwrite's core:
- full multi-agent orchestration internals
- full research automation pipelines
- external workflow runtime assumptions
- mandatory LaTeX package stacks and venue-specific low-level formatting recipes
- full citation export tooling implementations

## Design Principle

cycwrite should absorb **capability patterns**, **quality standards**, and **routing intelligence** from relevant OpenCode skills.
It should not become an unreadable mega-skill that duplicates the full internals of those skills.
