# Review Routing and Gates

## Purpose
This file strengthens cycwrite's review-article intelligence by absorbing the strongest ideas from the review-workflow materials.

## 1. Review Types

### Narrative / integrated review
Use when the user needs a field synthesis with thematic integration and argument.

### Scoping review / evidence map
Use when the user first needs to map what exists, where the clusters are, and where the gaps are.

### Systematic evidence review
Use when the question is narrow enough for explicit inclusion/exclusion logic and higher reproducibility.

### Umbrella review
Use when the core evidence base is already composed of systematic reviews or meta-analyses.

### Critical / theoretical review
Use when the literature is concept-heavy, theory-heavy, or historically structured rather than purely empirical.

## 2. Mandatory Routing Questions
Before drafting a review, answer:
1. What question is the review trying to answer?
2. Is the review mapping evidence or defending a stronger conclusion?
3. Can inclusion/exclusion criteria be defined clearly?
4. Is the core evidence primary research, secondary synthesis, or theory/classic texts?
5. What output level is realistic: full review, evidence map, framework memo, or partial synthesis?

## 3. Evidence Gates
A submission-grade review should not be claimed unless the user has, or can reasonably build:
- a stable scope boundary
- a visible corpus or document pool
- a chosen body framework
- evidence grouped into defensible modules
- at least a provisional claim-to-evidence map

If these are missing, cycwrite should downgrade the output to one of:
- review outline
- evidence map
- framework memo
- literature synthesis notes
- claim-to-evidence preparation table

## 3B. Output-Level Ladder
Choose the strongest honest deliverable level rather than defaulting to a polished review draft.

### Level 1 — submission-grade review draft
Use only when the project has:
- a stable, review-appropriate scope
- a visible and reasonably mature corpus
- at least provisional screening / inclusion logic
- a chosen body framework
- a defensible claim-to-evidence map

### Level 2 — review-grade evidence synthesis
Use when the corpus is already meaningful and clustered, but the manuscript is not yet safe to frame as a polished submission draft.

### Level 3 — evidence map / scoping output
Use when the most honest contribution is to show what exists, how it clusters, and where the gaps sit.

### Level 4 — framework memo / outline
Use when the writing logic is becoming clear but the corpus, coverage, or screening record is still weak.

State the chosen level explicitly when the user is asking for a review article.

## 4. Framework Selection
Common review body frameworks:
- time evolution
- thematic modules
- methodological routes
- application scenarios
- controversies and disagreements
- mechanism -> evidence -> limitation
- problem -> solution

Choose the framework that best explains the literature, not the one that merely looks elegant.

## 5. Reproducibility Awareness
When the user wants a stronger review type, cycwrite should encourage explicit recording of:
- search terms / query logic
- data sources
- date limits
- inclusion / exclusion criteria
- screening logic
- extraction categories
- quality or bias assessment if applicable

## 5B. Gate-Tier Questions
Before calling a review submission-grade, check:
- does each major body section have enough core evidence to avoid single-paper overreach?
- are recent and foundational sources both represented where that balance matters?
- are exclusion reasons or downgrading reasons recoverable if challenged later?
- does the chosen review type match the strength of the available record?

If the answer is no, lower the output level instead of smoothing over the gap with confident prose.

## 6. Strong-Conclusion Rule
Do not produce strong causal or field-level conclusion language in a review unless the evidence base obviously supports it.

Preferred fallbacks:
- "the current evidence suggests"
- "the literature appears to converge on"
- "available studies indicate"
- "evidence remains limited / heterogeneous / uneven"
